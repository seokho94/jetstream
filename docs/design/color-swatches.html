Jetstream색상 시안 · State badges & Current hues

다크 “인텔리전스 터미널” 테마(§3 토큰) 위에서 본 4개 모멘텀 상태 배지와, 그와 충돌하던 current 색상의 원본(스펙 §3) ↔ 제안(충돌 해소) 비교입니다. current 재배정 헥스는 CANON §14 R11로 확정되었습니다. 상단 무채색 시뮬레이션으로 색을 제거해도 아이콘+라벨로 상태가 구분되는지(색 단독 인코딩 금지) 확인하세요.

1 · 모멘텀 상태 배지 (4 states · 예약색)

각 배지는 색 + 아이콘 + 라벨을 항상 함께 씁니다. 이 4색은 color_registry.is_reserved=true로 예약되어 current 색에 쓰지 않습니다.

2 · 예약 색 (시스템 — current 금지)

상태 배지 4색 + 브랜드 틸(크롬·digest) + 상태 헬퍼(up/down). current hue는 이들과 같을 수 없습니다.

3 · Current 색상 — 원본 vs 확정

스펙 §3의 6색 중 3색이 상태 배지색과 동일(ai-governance=amber=rising, cost-of-living=coral=peaking, middle-east=steel=cooling)하고, energy=teal은 브랜드 틸과 충돌합니다. 제안은 이 4색을 구분되는 hue로 재배정합니다.

원본 (스펙 §3) — ⚠ 충돌 4

확정 (해소) — ✓

4 · 보드 미리보기 — 충돌이 실제로 보이는가

랭킹 행은 current 이름(=그 색) 옆에 모멘텀 배지가 옵니다. 원본에서는 이름색과 배지색이 겹쳐(예: AI governance 이름=amber, Rising 배지=amber) 구분이 흐려집니다. 제안에서는 분리됩니다.

원본 (충돌)

확정 (해소)

● 이름색 = current hue▮ 좌측 틱 = current hue배지 = 모멘텀 상태

5 · 상세 화면 헤더 (color = layer)

각 current 상세는 자기 hue로 테마링됩니다. 제안 색으로 헤더 밴드 + 상태 배지를 함께 보았을 때의 모습입니다.

확정 / 대기: 4개 상태 배지색은 스펙 §3·CANON §2의 LOCK 값입니다(#F5A524 / #FB7A50 / #7C9CC0 / #9BA3AF). current 재배정 헥스는 확정(CANON §14 R11) — ai-governance #C46BD8 · cost-of-living #E86A8E · energy #9CCB3B · middle-east #5C6BC0. 신규 current 배정 시에만 ΔE·WCAG 비텍스트 3:1·colorblind QA 게이트를 적용합니다.
